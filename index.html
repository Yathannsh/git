Git Workflow

  • What is Git?

    Git is a version control system gifted by Sir, Linus Torvlad to the open source world. Git lets you track smallest of changes you make in your projects or repositories as they are known in git lingo. To use git you will first need to download a git client.

  • Repositories in git

    For git to track changes in a repository it first has to be informed about its presence. Navigate to the folder where you already have a project which you want git to track or to the folder where you wish to make a new repository.

    - Right click in the folder where your repository is present and, select 'git bash here'. Follow this if you aren't comfortable with navigation in the command line.
    - Now to declare a repository or initialize it in the git lingo, type

    $git init

    You could also clone a repositorty by the following command

    $git clone <Repo directory>

  • Git branch?

    Now that you have a projet to work on its time to make changes, while making changes such as adding a new feature its always recommended to not link them with your main project or what in the git lingo is called the master branch. This is the reason git uses branching.
    To checkout the branch you are currently on:

    $git branch
    You will get output which will tell you you are on the master branch as you haven't created any branch yet.

    $git branch
    *master

    The asterix symbol indicates the branch you are currently on. Let's create a new repository. Type git branch and the name you want to give the branch

    $git branch new_branch
    Congratulations you just created a new branch!

  • Git checkout?

    Now that you know how to create a new branch let's learn how to switch between branches. First check the available branches.

    $git branch
    *master
    new_branch

    Git displays the available branches. Notice the new_branch you created previously, let's switch to that branch.

    $git checkout new_branch
    This will switch your repositories branch to new_branch. Let's see if it worked

    $git branch
    master
    * new_branch
    The change in asterix position confirms the switch.

  • Git status?

    Whenever you make a change to a repository, git tracks it. For the sake of testing create a new text file in your project with the name 'text'. Now as this is a new addition to the git repository, git will recogines this.

    Type $git status in the command line, and you will recieve the following output.
    $git status
    Untracked files:
    text.txt/

    To learn how to add these changes to the staging area proceed to the next accordin.

  • Git add?

    In the git workflow, after one makes changes to a working directory like you just did after adding a flie, they neeed to add it to the staging area. To add a particular file to the staging area type:

    $git add <File Name>

  • Git commit?

    Now the files are in the staging area,to finally add them to your repository you will need to commit them. While making a commit you are always required to add a message otherwise the commit wont be successful. To make a commit do the following.

    $git commit -m "Short commit message"

    Congratulation, you just made your first commit!

  • Git push?

    Git Push is a command used for pushing your code online. After you have made your repository and you wish to make it public you need to have a git hub account to make it avaialble online.Make a github account if you don't have one and create a new repository online.

    After making an online repository, Github will provide you with it online refrence URL. You first need to configure the git remote or the place where git will push your code. To do so

    $git remote add "Repository URL"

    After you have added a remote,you can push your code to your online repository. This how you do it.

    $git push origin master

    This code will push the master branch to your online repo.

  • Git log?

    Git lets you track changes commit-by-commit, so if anything goes wrong you can go right back to the place where things worked right. This is where commit messages play a great role, while you will learn how to track commit if it weren't for commit messages all those commits wouldn't be so helpful.
    To view all the commits made use the following function.

    $git log

    This command will output a history of commit logs, which will look something like this.

    $git log
    commit aa2b2c34c063c94e
    Author: Your_Name
    Date: Sat Dec 19 11;22;08 2015 +0530
    Your commit Message

  • Git rebase?

    Git rebase is a function used more in open source collaboration projects. Let's say you and your friend are working on a project simultaneously. Your friend makes a change in the project by adding a new feature while you as well add the same feature in a different way.

    This results in a conflict; however, if you both sit and decide and choose for one feature out of the two to be integrated, this problem can be solved by rebasing the master.
    So if your friend's feature is what you think is better for the project you will have to rebase, and this is how you do it.

    $git checkout the_new_feature
    $git rebase master

    The first line of code above navigates to the branch you'd like to rebase your master from. Then the master is rebased to that branch. Rebasing basically re-writes the project history by creating brand new commits for each commit in the original branch.

  • Git Stash?

    Git Stash is know as the dirt handler of Git. For instance if you are working on a project and things get messy and you wish to switch branches and leave your work as it is instead of committing them to your repository,git stash is the saviour. Git stash is always used on a semi-eddited file, we will call that file index.html just for the sake of this tutorial.
    This is how git stash is used:

    $git stash
    Saved working directory and the state index stat\
    WIP on master: 049d078 added the index file"
    HEAD is now at 049d078 added the index file
    (To restore them type "git stash apply")

    Typing the git stash command will stash the file for you, which you can then apply to your branch anytime you need to.

  • Git reset?

    It is common to mess up in life and writing code is no exception. One can mess up a lot of times, but git save you again with its reset function. With git reset function you can go back to the place where everything worked well.

    First you need to know where things worked well. Use git log to know that. If it all worked well till the last commit this is what you do:

    $git reset

    If this doesn't you wil need to know where things did go well by git log and copy the refrence ID of the commit, and use the follwoing command.

    $git reset -- hard ><Commit refrence>



    This is it! Now you can work smoothly from the place where all went well.
